## 데이터 모델 이해
# 데이터 모델링이란
1. 정보 시스템을 구축하기 위한 데이터 관점의 업무 분석 기법
2. 현실의 데이터를 약속된 표기법으로 표현하는 과정
3. 데이터베이스 구축을 위한 분석 및 설계의 과정

# 데이터 모델링의 특징
추상화, 단순화, 명확성

# 데이터 모델링의 관점
데이터, 프로세스(업무), 상관(처리 방식)

# 데이터 모델링의 유의사항
유연성, 유일성, 일관성

# 데이터 모델링의 3단계
1. 개념적(추상) - 전사적 관점, ERD 작성, 추상화 수준 높음
2. 논리적(정규화) - 정규화 수행, 재사용성 높음, 특정 DB에 종속되지 않음
3. 물리적 모델링(DB) - 성능, 저장소, 반정규화(De-normalization) 고려, 실제 테이블 생성

# 데이터 베이스 스키마 구조
외부(뷰)-개념(통합된 사용자)-내부스키마(물리)

# 데이터 독립성
논리적, 물리적

# 데이터 모델링의 3요소
엔터티, 관계, 속성

# ERD 작성 순서
1. 엔터티 도출
2. 배치
3. 관계설정
4. 관계명 기술
5. 관계 차수 설정
6. 선택사양 기술

## 엔터티 (Noun)
# 엔터티란
업무에서 관리해야 하는 데이터의 집합, 단수명사, 인스턴스의 집합

# 엔터티 특징
1. 업무에서 필요로 함
2. 유일한 식별자
3. 2개 이상의 인스턴스 집합
4. 업무 프로세스에서 이용
5. 2개 이상의 속성
6. 관계를 갖고 있음

# 엔터티 유무형 분류
유형, 무형

# 엔터티 시점에 따른 분류
기본, 중심, 행위

# 엔터티 명명 규칙
1. 협업에서 사용되는 용어
2. 약어 지양
3. 단수 명사
4. 유일성 보장
5. 의미 명확

## 속성
# 속성이란
업무에서 필요로 하는 인스턴스에서 관리하고자 하는 의미상 더 분리되지 않는 최소의 데이터 단위

# 속성의 특징
1. 업무에서 필요
2. 주 식별자에 함수적으로 종속(기본키 변경시 속성도 변경)
3. 1개의 속성은 1개의 속성 값을 가짐
4. 속성도 집합

# 속성 특성에 따라 분류
기본, 설계, 파생 속성

# 속성 분해 가능 여부
단일, 복합, 단일값, 다중값 속성

# 속성의 도메인이란
속성이 가질 수 있는 값의 범위를 뜻 함

## 관계 (Verb)
# 관계의 표기법
관계명, 관계차수(많으면 삼발 표시), 관계 선택사양(필수=I, 선택=O 로 표시)

# 관계의 ERD
존재, 행위 (표기 구분 안 함)

# 관계의 UML
연관관계(실선, 멤버변수), 의존 관계(점선, 파라미터)
```
식별관계(실선)
강한 연결관계, 자식 식별자의 구성에 포함, 부모 엔터티에 종속
```
```
비식별관계(점선)
약한 연결관계, 자식 일반 속성에 포함)
```

## 식별자
# 식별자란
엔터티를 대표할 수 있는 유일성을 만족하는 속성(자주 이용되는 속성, 명칭이나 내역 이름은 X)

# 식별자 특징
1. 유일성
2. 최소성
3. 불변성
4. 존재성
```
대표성      |주식별자    |보조식별자
스스로 생성  |내부식별자  |외부식별자
속성의 수    |단일식별자  |복합식별자
대체 여부    |본질식별자  |인조식별자
- 내부 식별자 = 엔터티 내부에서 스스로 생성
- 외부 식별자(FK) = 타 엔터티로부터 상속받아 온 식별자
- 본질 식별자 = 업무 프로세스에 의해 만들어지는 식별자
- 인조 식별자 = 업무족으로 존재하지 않지만 관리 편의상 만든 식별자(예:일련번호)

```
## 정규화(NF)
# 제 1정규형
모든 속성은 반드시 하나의 값을 가져야 한다 = 원자성
# 제 2정규형
엔터티의 일반 속성은 주식별자 전체에 종속이어야 한다 = 부분함수종속성
# 제 3정규형
엔터티의 일반속성 간에는 서로 종속적이지 않아야 한다 = 이행함수종속성
# 보이스코드 정규형란
후보키가 기본키 속성 중 일부에 함수적 종속일 때 다수의 주식별자를 분리

# 정규화를 하면 성능이 무조건 좋아지는가?
정규화시: 입력/수정/삭제 성능은 향상, 조회(SELECT)시 JOIN이 증가해 조회 성능은 저하 될 수 있음
이때 조회 성능을 위해 수행하는게 반정규화(역정규화)

## 관계와 조인의 이해
1. 조인이란 식별자를 상속하고 상속된 속서을 매핑키로 활용해 데이터를 결합하는 것
2. 부모의 식별자를 자식의 식별자에 포함하면 식별관계, 부모의 식별자를 자식의 일반속성으로 상속하면 비식별관계
   (비식별관계에서 조인 많이 발생 함)
3. 관계를 맺는다는 것은 식별자를 상속시키고 해당 식별자를 매핑해 데이터를 결합하는 것

## Null 속성 이해
# Null의 특성
1. 아직 정의되지 않은 값으로 0이나 ''이 아님
2. NOT NULL 또는 PRIMARY KEY 외 모든 데이터 유형에 포함 가능
3. NVL, ISNULL로 다른 결과값 얻음
4. 집계 함수에선 제외

# Null의 연산
1. NULL값과의 연산은 Null을 리턴 함 EX: NULL + 10 = NULL (산술 연산)
2. 모든 비교는 알 수 없음(Unknown) 리턴
   NULL = NULL 은 False(Unknown) 반드시 IS NULL을 사용해야 함
3. 집계함수는 Null을 제외하고 계산
   SUM(컬럼)은 NULL을 제외하고 계산하지만, COUNT(*)는 NULL을 포함한 전체 행 수를 셈

## 본질과 인조 식별차 
1. 인조 식별자는 대체로 본질 식별자가 복잡한 구성을 가질 때 만들어짐
2. 인조 식별자를 사용하면 중복 데이터를 막기 어려워짐
3. 인조 식별자를 사용하면 본질 식별자를 사용할 때와 비교해 추가적인 인덱스가 필요해짐
4. 인조 식별자는 단점도 존재해 꼭 필요한 경우에만 사용하는 것이 바람직하다.
